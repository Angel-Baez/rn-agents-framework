# Agent Catalog

Complete reference for all 18 agents in the RN Agents Framework.

## Core Agents

### orchestrator
**Role**: Central router and coordinator
**When to use**: Entry point for all requests, routing to specialized agents
**Keywords**: orchestrate, route, coordinate, delegate

### product-manager
**Role**: Requirements gathering and user story creation
**When to use**: Defining features, acceptance criteria, prioritization
**Keywords**: user story, requirements, feature, backlog

### solution-architect
**Role**: High-level technical decisions and ADRs
**When to use**: Architecture decisions, technical trade-offs
**Keywords**: architecture, ADR, design, technical decision

### code-reviewer
**Role**: Code quality and best practices enforcement
**When to use**: Pull request reviews, code quality checks
**Keywords**: review, quality, best practices, standards

### documentation-engineer
**Role**: Technical documentation
**When to use**: Writing API docs, guides, READMEs
**Keywords**: documentation, docs, guide, readme

## Mobile Development Agents

### mobile-architect
**Role**: UI/UX design and React Native components
**When to use**: Building screens, components, navigation, styling
**Keywords**: component, screen, UI, navigation, styling, NativeWind

### backend-architect
**Role**: APIs and Supabase configuration
**When to use**: Creating APIs, authentication, database queries
**Keywords**: API, endpoint, Supabase, backend, authentication

### data-engineer
**Role**: Database schema and data modeling
**When to use**: Designing schemas, storage strategies, migrations
**Keywords**: database, schema, storage, migration, data model

## Mobile-Specific Agents (NEW)

### native-modules-engineer
**Role**: Expo SDK modules and native APIs
**When to use**: Camera, location, push notifications, permissions

### performance-optimizer
**Role**: Performance optimization
**When to use**: Bundle size issues, slow renders, memory leaks
**Keywords**: performance, slow, optimization, memory, bundle size

### eas-specialist
**Role**: EAS Build, Submit, and Update
**When to use**: Building apps, deploying to stores, OTA updates
**Keywords**: EAS, build, deploy, submit, OTA, update

### offline-architect
**Role**: Offline-first architecture
**When to use**: Implementing offline features, sync strategies
**Keywords**: offline, sync, network, connectivity
